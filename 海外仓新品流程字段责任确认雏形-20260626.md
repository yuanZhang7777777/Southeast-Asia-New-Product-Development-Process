# 海外仓新品流程字段责任确认雏形

> 日期：2026-06-26  
> 用途：下午和领导确认“哪些表、哪些字段、由谁填写、哪些由系统预填”。  
> 状态说明：`已确认` 表示已按当前口径先定；`待确认` 表示下午需要领导拍板。

## 1. 当前先定的总原则

1. 系统能带出的字段不让人重复填：SKU、国家、仓库、成本、体积、销量、库存、店铺、Item、历史单销、竞品基础数据等，优先从钉钉在线表 / ERP清洗结果 / 现有货盘表预填。
2. 人工只填需要判断的字段：是否认领、认领单销、不认领理由、销售反馈总结、备货数量、二次调研结论、销售策略、复盘总结等。
3. 分配在认领之前：先由练玉君按主SKU整组分配“主销售员”，同一主SKU下所有子SKU不拆分，分配目标是各销售员子SKU数量尽量均衡。
4. 销售认领单销是日销 `n`，默认首批备货量按 `n * 30天` 生成。
5. 不认领必须留痕：销售需填写不认领理由，并在销售反馈总结里说明调研实情、为什么不认领，必要时附截图或链接。
6. 主管/组长/销售员各自进平台只看自己的待办：组长看分配，销售看认领和调研，主管看复核和异常，运营/销售看到货后操作和推品。

## 2. 整体链路雏形

```text
选品来源
  ├─ 开发部门开发新品认领-反馈
  ├─ 财根团队开发新品认领-反馈
  ├─ 直发热销转海外仓销售确认
  └─ 销售选品备货海外仓-销售调研
        ↓
组长分配主销售员 / 销售认领 / 主管复核
        ↓
海外仓备货申请表
        ↓
采购 / 供应链核价 / 到仓
        ↓
【到货通知】认领制度到货子SKU
        ↓
【市场监控】海外仓精品市场调研
        ↓
【推品】海外仓精品流程
        ↓
销售精品成功率 / 复盘 / 后续补货、缺货、清仓、调拨
```

## 5. 操作台页面雏形

第一版平台可以先做成 4 个角色视角：

| 角色 | 进入后看到什么 | 主要动作 |
|---|---|---|
| 练玉君/组长 | 待分配主SKU清单、每个销售员当前子SKU数量 | 按主SKU整组分配主销售员，查看均衡度 |
| 销售员 | 我的待认领、我的待调研、我的到货待操作、我的推品复盘 | 看系统预填信息后填写认领/不认领、单销、原因、调研、策略 |
| 销售主管/部门负责人 | 本期认领进度、无人认领、低单销、异常反馈 | 复核、退回、汇总确认 |
| 运营/店铺负责人 | 到货待确认、库存同步、刊登覆盖、推品动作 | 确认库存/刊登/调价/策略执行 |

MVP 页面顺序建议：

1. `我的待办`：按角色过滤，只展示当前人要处理的行。
2. `分配台`：练玉君使用，按主SKU整组分配，显示子SKU数量均衡度。
3. `认领决策页`：销售先看产品信息、竞品、利润、库存/历史数据，再填是否认领。
4. `备货申请页`：从认领结果自动生成草稿，确认备货数量、国家、仓库。
5. `到货操作页`：到货后确认认领销售、销售操作、库存/刊登/调价状态。
6. `二次调研页`：展示首次调研和当前市场变化，销售/运营补二次结论。
7. `推品复盘页`：填策略、优化动作、总结；数据指标系统自动带出。
