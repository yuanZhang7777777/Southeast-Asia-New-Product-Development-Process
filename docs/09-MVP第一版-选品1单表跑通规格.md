> 日期：2026-06-29  
> 状态：实施规格，交 Codex 实现，Claude 验收  
> 目标：用 **选品1（海外仓开发部门开发新品认领-反馈）单张源表**，跑通 MVP 前段：`入池 → (分配/自领) → 销售认领 → 主管审核 → 出可备货清单`。

## 0. 关键口径（来自已确认结论）

- 框架 = 两段 + 一桥；本次只做 **前段，止于“出可备货清单”**。
- 角色：销售 = 运营；主管 = 运营负责人。
- 入池粒度：**子 SKU 为记录，主 SKU 分组**。
- 备货量 = **认领单销 × 30**（销售自己填认领单销）。
- 不认领原因 = 自定义文本。

## 3. 入池规则

- 子 SKU(J) 为一条记录；主 SKU(H) 用于分组。
- **去重**：`source_file + source_sheet + source_row`（现有唯一约束 `uq_opportunity_source_row`），importer 可重复跑、幂等。
- 每行存一份整行快照到 `source_record_snapshot` / `opportunity.snapshot`。
- **跳过无效行**：H 主SKU 或 J 子SKU 为空、或整行为小计/空行的，不入池。

## 4. 跑通前段（用现有骨架）

1. **入池**：importer 落库 → 机会列表（按主 SKU 分组展示，站点过滤）。
2. **分配/自领**：CD 有值则预分配；否则开放池，销售自领。
3. **认领**：销售在平台填 CE/CF/CC/CG（源表有值则预填，状态仍以平台提交为准）。
4. **审核**：平台生成 review 任务，主管=运营负责人 点 通过 / 驳回 / 退回补充。
5. **减法（务必一起改）**：审核通过 **不再自动按 ×30 建 `stocking_request`**（现 `backend/app/services.py:182` 的 `create_stocking_draft_from_claim` 要改）——改成把机会标记为“可备货”，进入第 5 节导出。

## 5. 出可备货清单（导出 Excel，对齐《海外仓备货申请表》）

来源 = 审核通过的机会。列：

| 清单列 | 取值 |
|---|---|
| 操作状态 | 默认“未操作” |
| 时间 | 审核/导出时间 |
| 备货类型 | 首次备货 |
| 选品数据源 | `选品1-开发部门认领反馈 + 批次sheet名` |
| 销售员 | CD 主销售员 / 认领人 |
| 主SKU / 子SKU | H / J |
| 站点 | A |
| 认领单销 | CF |
| **备货量** | **CF × 30** |
| 备货国家 / 仓库 | 留空（待业务/外部定） |
| 成本价 / 单个体积 / 货值 | **留空**（本表已去掉成本列 M:Y、AW:BN；外部备货时补） |
| 补货原因 | 空（首次） |
| ★是否需要开品邮件 | 空（平台后续标记） |
| ★开品邮件状态 | 空 |

## 6. 验收标准（Claude 审）

1. 导入 `开发0623期`：入池机会数 = 有效数据行数；重复跑不产生重复（去重生效）。
2. 入库字段只来自 `A:L / Z:AN / AO:AV / CC:CH`；`M:Y`、`AW:CA`、`CB` 一列都没进库。
3. 子 SKU 为记录、主 SKU 分组、站点取自 A 列正确。
4. 认领→审核走得通；**审核通过不再自动建 stocking_request**。
5. 导出可备货清单 Excel：备货量 = 认领单销 × 30；成本/货值列留空；含开品邮件两列。
6. 无效/空行被正确跳过。
