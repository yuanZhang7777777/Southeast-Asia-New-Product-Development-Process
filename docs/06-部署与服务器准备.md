# 部署与服务器准备

> 日期：2026-06-29  
> 服务器：`101.132.26.138:2323`  
> 状态：作为预生产基础环境和运维跳板，不直接承诺生产容量。

## 1. 本机 SSH 别名

使用别名：

```powershell
ssh hz-new-product-preprod
```

配置模板见：

```text
deploy/ssh-config.example
```

不要把 root 密码、钉钉密钥、ERP 密钥写入项目文件。已暴露过的密码建议轮换。

## 2. 服务器资源基线

当前探测结果：

| 项 | 当前值 | 结论 |
|---|---:|---|
| CPU | 2 vCPU | 可做轻量预生产 |
| 内存 | 约 3.5 GiB + 2 GiB swap | 低于推荐 4C8G |
| 系统盘 | 40 GiB，总剩余约 28 GiB | 低于建议 60-100 GiB |
| Docker | Docker 26.1.3 / Compose v2.27.0 | 已安装并设置开机启动 |

结论：当前机器适合跑基础 Docker、API 骨架、Hermes 测试和小规模预生产验证。若 100-200 人正式使用，建议迁移或扩容到至少 `4C8G / 100GB SSD`，生产建议 `4-8C / 16GB / 200GB SSD` 起步。

Docker 安装说明：Alibaba Linux 3 的 `dnf install docker` 会安装 `podman-docker` 兼容层，不满足本项目 Docker Compose 基线；当前脚本改为使用阿里云 Docker CE 镜像源安装真实 Docker。

镜像源说明：服务器访问 Docker Hub 和 PyPI 官方源不稳定，已在 Docker daemon 配置镜像加速；后端 Dockerfile 默认使用清华 PyPI 镜像源。

## 3. 服务器目录

初始化脚本会创建：

```text
/opt/hengzhe-new-product/app
/opt/hengzhe-new-product/data
/opt/hengzhe-new-product/backups
/opt/hengzhe-new-product/hermes
/opt/hengzhe-new-product/logs
```

## 4. Compose 服务

`docker-compose.yml` 包含：

- `api`
- `frontend`
- `worker`
- `scheduler`
- `postgres`
- `redis`
- `reverse-proxy`
- `hermes`（profile 启动）

本地或服务器启动：

```bash
docker compose up -d --build
```

Hermes 适配器单独启动：

```bash
docker compose --profile hermes up -d --build hermes
```

## 5. 当前预生产状态

已在服务器路径启动基础 Compose 栈：

```text
/opt/hengzhe-new-product/app
```

服务器本机验证：

```bash
curl http://127.0.0.1:8080/api/health
```

返回：

```json
{"status":"ok","app":"Hengzhe New Product Workflow","environment":"preprod"}
```

当前外网访问 `http://101.132.26.138:8080/api/health` 从本机测试超时。服务器 OS 防火墙未拦截，Docker 已监听 `0.0.0.0:8080`，需要在阿里云安全组放行 8080，或后续绑定域名并改为 80/443。

## 6. 安全和备份

- `.env` 不入库。
- 生产密码使用服务器环境变量或 secret 文件。
- PostgreSQL 后续增加每日备份任务，保留 14-30 天。
- 管理端口优先走 SSH tunnel。
- 外网只开放 SSH、HTTP/HTTPS 必要端口。
