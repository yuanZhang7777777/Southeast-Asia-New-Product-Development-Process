## 1. 项目一句话定位

本项目要做的是“东南亚海外仓新品流程协同平台”。

它不替代 ERP、销售系统、运营系统，**也不替代采购、开发、供应链部门**，而是站在 **运营部门** 视角，在现有系统和钉钉在线表之上做一个流程协同层。平台拥有“前段·选品决策”和“后段·到货运营”两段主流程，把分配、认领、审核、刊登、监控、复盘变成可追踪的系统流程；中间的备货/采购作为外部桥只做衔接（导出清单 + 登记状态 + 到货通知回流）。

## 2. 当前共识

1. 第一版不要一次性替换所有钉钉在线表操作习惯。
2. 第一版应尽量贴近原来的表格逻辑，先解决责任人、状态、截止时间、提醒和进度追踪。
3. 酷应用作为钉钉内的处理入口。
4. 钉钉待办用于必须处理的任务，例如销售认领、主管审核；**备货/采购属于外部桥，不建运营待办**。
5. 钉钉工作通知用于提醒、异常、汇总和升级。
6. ERP、销售系统、运营系统已有的数据尽量同步过来，不让人重复填。
7. 人工只填写需要业务判断的字段，例如是否认领、认领单销、不认领原因、审核意见、调研结论、复盘总结。
8. **角色口径：销售 = 运营（同一拨人）；主管 = 运营负责人（同一角色）。**
9. **框架是“两段 + 一桥”**：前段平台主流程、外部桥只衔接、后段平台主流程；MVP 只做前段。

## 5. 当前主流程（两段 + 一桥）

```text
【前段·选品决策｜平台主流程·MVP】
  待认领数据(入池) → 分配 / 销售自领 → 销售认领 → 主管审核 → 出“可备货清单”
        │  导出可备货清单 +（条件性）开品邮件状态标记
        ▼
【外部桥｜非平台待办，只导出 + 登记状态】
  开发备货 / 销售备货 / 供应链核价 / 采购下单 / 开船·到港·到仓
        │  到货通知回流
        ▼
【后段·到货运营｜平台主流程·后续阶段】
  到货承接(到货通知) → 刊登 → 库存/售价/链接/ItemID/店铺/站点确认
   → 二次调研 → 推品(周度复盘循环 ≥4周) → 四周总结 → 后续动作
        ┄ 旁路：随时补货、滞销清仓
```

> 关键修正：**备货/采购不是平台主流程节点，是外部桥**；**到货刊登是后段重点**，不要再按旧版排在备货之后当“附属”。

其中已识别的主要分支：

- 销售认领：认领填单销 / 不认领填理由（不认领原因为自定义文本）
- 主管审核：通过 / 驳回 / 退回补充（主管 = 运营负责人）
- 后段二次调研：外部老品热销监控 / 老品销量下跌 / 销售自主调研
- 后段推品：缺货监控 / 广告费监控
- 后段复盘：周度复盘循环，每轮可重设“目标单销”，达成判定对“最后一次目标”

## 6. 和技术人员沟通时的推荐说法

可以这样描述：

> 我们这次不是单纯把在线表换成网页表单，而是做一个新品流程协同层。我们是运营部门，不替代采购/供应链；这套系统是“两段主流程 + 一座外部桥”。  
> 第一版（MVP）只做前段：把待认领、分配、销售认领、主管审核串起来，审核通过后产出“可备货清单”交给外部备货采购。备货/采购走外部桥，平台只导出清单、登记状态；到货后由“到货通知”回流，再接后段的刊登、监控、四周总结。  
> 已有系统能提供的数据尽量同步，人工只补判断类字段。钉钉酷应用作为入口，必须处理的事走待办，异常和汇总走工作通知。

## 7. 下一轮重点要讨论的问题

框架、MVP 范围、入池源表、入池粒度、角色口径已确认（见上）。仍需逐表确认：

1. 每张源表的字段：哪些进系统、谁填、是否必填、是否写回在线表（用字段确认操作台逐行过）。
2. 认领必填字段、认领单销周期（日销？）、首批备货是否默认按 `单销 × 30`。
3. 可备货清单最终字段（对齐《海外仓备货申请表》）。
4. 开品邮件：哪些货需要、模板字段、收件人（领导/老板）。
5. 平台提交后是否写回钉钉在线表。
6. 后段（到货后）各字段：到货触发点、ItemID/链接回填来源、二次调研/复盘字段——走到后续阶段再定。

## 8. MVP 建议范围

第一版只做 **前段最小闭环，止于“出可备货清单”**：

```text
入池（5 张源表：开发部门认领-反馈 / 财根团队认领-反馈 / 直发热销确认 / 销售选品调研 / 销售反推表）
  → 分配 / 销售自领（按主 SKU 整组；分配人后定）
  → 销售认领（是否认领 · 认领单销 · 不认领原因）
  → 主管审核（通过 / 驳回 / 退回补充）
  → 产出“可备货清单”（对齐《海外仓备货申请表》，先导 Excel；含开品邮件状态标记）
```

入池粒度：子 SKU 为记录 + 主 SKU 分组。
提醒第一版只做 3 条：入池/分配后催认领、认领后催审核、逾期升级。
备货监控、到货承接、刊登、二次调研、推品、四周总结全部后置为后续阶段，不进 MVP。
