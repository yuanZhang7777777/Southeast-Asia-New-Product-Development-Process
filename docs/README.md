# 东南亚新品流程项目文档入口

> 本目录是后续需求讨论、开发实现、会话切换时的共同上下文。  
> 新会话或新开发人员接手前，优先阅读本文件，再阅读下方两份核心文档。

> 🧭 一句话框架（2026-06-29 运营视角校正）：本平台是 **“两段主流程 + 一座外部桥”**。
> 平台只拥有 **前段（入池→分配→认领→审核→出可备货清单）= MVP** 与 **后段（到货承接→刊登→监控→四周总结）**；
> 中间 **备货 / 采购 / 供应链 / 下单 = 外部桥**，平台只导出清单 + 登记状态 + 等到货通知回流，不建运营待办。
> 角色口径：销售 = 运营、主管 = 运营负责人。

## 文档维护规则

1. 讨论出新规则时，先更新 `01-需求文档-东南亚新品流程.md`。
2. 功能实现、联调、上线、废弃时，更新 `02-功能实现状态.md`。
3. 不确定的内容不要写成已确认结论，统一放到“待确认问题”。
4. 已确认规则如果被推翻，要在变更记录中说明日期、原因和新口径。
5. 文档里的“建议表/模块名”不等于最终数据库表名，最终实现前需要再做技术命名。
